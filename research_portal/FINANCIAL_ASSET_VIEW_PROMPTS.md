# 金融资产观点提取 Prompt

## 推荐用法

采用两阶段流程：

1. 每篇 PDF 单独运行“Prompt A”，形成完整、可追溯的观点卡。
2. 将同一批观点卡交给“Prompt B”，去重、比较和压缩成精练的分析报告。

这样可以同时满足“提取尽可能充分”和“最终输出精练”。不要直接把大量 PDF 一次性摘要，否则容易遗漏正文、配置表、Top Picks、图表注释和具体期权条款。

## Prompt A：单篇研报观点完整提取

```text
你是一名买方跨资产策略研究员。请完整阅读所提供的整篇卖方研报（包括首页摘要、正文、图表、表格、附录中的当前配置/Top Picks），提取其中所有与金融资产定价或交易有关的观点，形成结构化“观点卡”。

目标：底层提取要充分、准确、可追溯；表达要高密度、无重复。不要写泛泛的全文摘要。

【输入】
- 文件名：{{file_name}}
- 报告元数据（如有）：{{metadata}}
- PDF/逐页文本：{{pdf_content}}

【第一步：识别报告】
提取机构、标题、发布日期、作者/团队、报告类型、主要市场与主要议题。报告类型从以下选取：宏观、跨资产、区域/指数策略、行业策略、个股、量化/资金流、衍生品、事件点评、其他。

【第二步：逐项识别资产观点】
扫描以下对象，不得只看标题和首页：
- 股票：国家/地区指数、行业、风格/因子、主题篮子、个股；
- 利率：国债、久期、收益率、曲线、通胀预期；
- 外汇、商品、信用、波动率；
- 期货、期权、价差及其他衍生品结构；
- 对资产有直接定价影响的宏观判断，如增长、通胀、央行与流动性。

每一个相互独立的观点建立一张观点卡，并回答：
1. 对象：具体是什么资产、市场、代码、期限或合约？
2. 结论：看多/看空/中性，绝对观点还是相对价值观点？相对谁？
3. 强度：强/中/弱。依据原文措辞判定，并附原文信号词，如 recommend、prefer、favor、OW、UW、should 等；不要自行提高确信度。
4. 时间：即期、战术（数日到3个月）、中期（3到12个月）、长期（1年以上）；若原文未说明，写“未说明”。
5. 状态：新观点、维持、加强、弱化、上调、下调、观察，或无法判断。
6. 核心逻辑：用“驱动因素 → 盈利/估值/供需/资金面影响 → 资产方向”的因果链表达。
7. 证据：最多保留3项最有辨识度的数据、估值、盈利预测、仓位/资金流或图表结论，必须带单位、口径和日期（原文有则保留）。
8. 催化剂：什么事件、数据或条件会推动观点兑现？预计何时？
9. 操作：买入、加仓、持有、减仓、卖出、做空、轮动、配对、对冲或期权结构；写明工具、资金来源腿/对手腿、入场条件、目标价、执行价、到期日、权利金、参考价和仓位建议。原文没给出的字段一律写“未给出”，不得补造。
10. 风险与证伪：哪些条件会使观点失效？区分一般风险与明确的止损/证伪条件。
11. 观点归属：必须从“本报告作者明确建议 / 本机构其他分析师观点 / 公司管理层或专家表态 / 客户或市场共识 / 资金流与仓位事实 / 你的合理推断”中选择。
12. 证据位置：标注 PDF 页码；若观点由多页共同支持，列出关键页码。

【真实性与归属规则】
- 严格区分“作者建议”与“市场行为”。例如“散户买入某股票”只是资金流事实，不能改写成作者建议买入。
- 严格区分作者观点、被引用的本机构其他分析师观点、公司管理层表态、客户反馈和你的推断。
- 只有原文明示时，才可写“建议买入/卖出/做多/做空”。由事实推导出的交易含义必须标为“推断”，且不得进入最高优先级建议。
- 不把免责声明、评级定义、评级分布或历史评级/目标价表误当成当前观点。当前评级只有在正文、首页或当前评级表明确出现时才提取。
- 同一资产若存在不同期限、不同情景或互相矛盾的观点，不要强行合并；分别保留并解释差异。
- 图表只提炼支持观点的结论和关键数字，不机械抄录整张表。OCR 或表格顺序可疑时标注“需核验”。
- 不使用报告发布日期之后的信息，不评价观点后来是否正确。
- 缺失信息写“未说明/未给出”，绝不猜测。

【优先级】
- P1：明确交易建议，且有较具体的工具、时点、目标或触发条件；
- P2：明确方向性/配置观点，但执行细节不完整；
- P3：重要前瞻判断、风险提示、资金流或仓位观察，没有直接操作建议。

【输出格式】

# 研报识别
一行输出：日期｜机构/团队｜标题｜报告类型｜主要市场/议题

# 核心结论
- 最多5条；每条只写一个结论，优先 P1/P2；避免与后表重复解释。

# 资产观点总表
用紧凑 Markdown 表格输出全部观点，每个独立观点一行：
优先级｜资产（代码）｜方向/相对关系｜期限｜状态｜一句话观点｜操作｜催化剂｜证伪/主要风险｜归属｜页码

# 重点观点展开
只展开 P1，以及最多5条最重要的 P2。每条严格使用以下格式：

## [资产]｜[方向]｜[期限]｜[优先级]
- 观点：一句话。
- 逻辑：驱动因素 → 传导机制 → 资产影响。
- 证据：最多3项关键数字/图表结论。
- 执行：工具；入场/触发；目标/到期；仓位/资金来源腿。未给出则明确写出。
- 催化与证伪：合计不超过3条。
- 归属与来源：[归属]；PDF p.X[, p.Y]。

# 观察项
只列 P3 中会影响后续交易判断的内容，最多5条。

# 信息缺口与核验
只列可能改变结论的缺失、OCR疑点或原文矛盾；若无则写“无重大缺口”。

【精练要求】
- 中文输出；保留标准英文资产名、ticker、指数名和期权条款。
- 先结论后理由；一句只表达一个意思。
- 删除作者履历、联系方式、免责声明、重复背景和无交易含义的行业常识。
- 同一事实只出现一次；重点展开引用总表而不复述整段背景。
- 除完整观点总表外，正文原则上控制在1200字以内。
```

## Prompt B：多篇研报综合成精练分析报告

```text
你是一名买方首席策略编辑。输入是多篇研报经完整抽取得到的“观点卡”。请在不丢失重要分歧、期限和操作条件的前提下，生成一份高密度的金融资产观点报告。

【输入】
- 分析截止日：{{as_of_date}}
- 覆盖范围：{{scope}}
- 单篇研报观点卡：{{view_cards}}

【处理流程】
1. 标准化资产名称、ticker、方向、期限和单位。
2. 合并同机构、同资产、同方向、同期限的重复观点；保留最新来源，并注明其他支持来源。
3. 同一资产在不同日期有变化时，明确写“上调/下调/转向/维持”，不要把旧观点当作并列现状。
4. 同一资产存在相反观点时，不做虚假平均；说明分歧来自期限、情景、作者团队或证据差异。
5. 严格保留观点归属。资金流、客户反馈、管理层表态不能升级为机构交易建议。
6. 按以下顺序排序：明确可执行性 > 原文观点强度 > 催化剂临近度 > 证据完整度 > 新鲜度。
7. 每个资产最终只保留：一句核心观点、一条因果链、最多3项证据、一个执行方案、主要催化剂和证伪条件。
8. 未被原文明确支持的组合建议可作为“综合推断”，必须单独标识，不能冒充原报告建议。

【输出格式】

# 一页结论
- 市场主线：最多4条。
- 最值得执行：最多5条，格式为“资产｜方向｜期限｜核心动作｜关键触发”。
- 最大风险：最多3条。

# 可执行观点清单
用紧凑 Markdown 表格，最多12条：
优先级｜资产｜方向/相对关系｜期限｜核心逻辑｜具体操作｜催化剂｜证伪条件｜最新来源（日期，页码）

# 重点观点展开
只展开排名前5的观点，每条不超过120字：
- 结论与操作
- 核心因果链及1至3项关键证据
- 催化剂与证伪条件

# 跨资产传导
最多5条，用“事件/变量 → 第一层资产 → 第二层资产”的格式，区分原文观点和综合推断。

# 分歧与变化
仅列真正会影响仓位的观点冲突或较前期变化，最多5条。

# 观察清单
列未来需要跟踪的数据、事件或价格触发条件，最多8条；每条注明对应资产。

【硬性要求】
- 中文输出，保留 ticker、合约与标准英文简称。
- 先给动作，再给逻辑；不写新闻式摘要。
- 不重复，不堆形容词，不使用“值得关注”等空话。
- 没有明确操作建议时，写“仅方向判断”或“仅观察”，不得自行生成交易。
- 所有具体数字、目标、执行价、到期日和引文结论必须可追溯到来源页码。
- 正文控制在1800字以内；如观点超过容量，只保留高优先级项目，并在末尾写“另有X条P3观察项未展开”。
```

## 人工使用时的简化版（单次直接分析 PDF）

```text
完整阅读所附研报，提取所有金融资产观点。先在内部逐页建立观点卡，再只输出压缩后的中文报告。

对每个观点必须识别：资产/代码、方向及相对关系、期限、观点强度、状态变化、核心因果链、最多3项关键证据、催化剂、具体操作条款、风险与证伪条件、观点归属、PDF页码。

严格区分：作者明确建议、本机构其他分析师观点、管理层/专家表态、客户共识、资金流事实和你的推断。不得把散户/机构资金流写成作者推荐，不得把免责声明或历史评级表当成当前观点；原文没有的入场点、目标、期限、仓位和止损不得补造。

按以下结构输出：
1. 核心结论：最多5条。
2. 可执行观点表：最多12条，字段为“资产｜方向｜期限｜一句话观点｜操作｜催化剂｜证伪｜归属｜页码”。
3. 重点展开：只展开最重要的5条，每条包含“逻辑链、关键证据、执行、催化与证伪”，每条不超过120字。
4. 分歧/变化：最多5条。
5. 观察清单：最多8条。

中文输出，保留 ticker 和合约条款；先动作后逻辑；不重复、不写泛泛摘要；正文控制在1800字以内。
```
